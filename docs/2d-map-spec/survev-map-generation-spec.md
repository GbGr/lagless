# Спецификация генерации карты в стиле Survev

Полное формализованное описание алгоритмов, параметров и порядка действий для воссоздания системы генерации карт из survev (battle royale, вид сверху).

---

## 1. Общие параметры карты

| Параметр | Значение | Описание |
|----------|----------|----------|
| `baseWidth` | 512 | Базовая ширина карты в игровых единицах |
| `baseHeight` | 512 | Базовая высота карты |
| `scale` | 1.19 - 1.28 | Множитель масштаба (зависит от кол-ва игроков) |
| `extension` | 112 | Добавка к размеру |
| `shoreInset` | 48 | Отступ берега от края карты |
| `grassInset` | 18 | Отступ травы от берега (ширина пляжа) |
| `gridSize` | 16 | Размер ячейки сетки |
| `seed` | random int | Сид для детерминированной генерации |

**Итоговый размер:**
```
width  = baseWidth  * scale + extension   // например: 512 * 1.1875 + 112 = 720
height = baseHeight * scale + extension
```

## 2. Палитра цветов (биом)

Каждая карта определяет набор цветов. Пример для стандартного биома:

| Слой | Цвет (hex) | Описание |
|------|-----------|----------|
| `background` | `#20536e` | Глубокий океан за пределами карты |
| `water` | `#3282ab` | Вода (океан у берега + реки) |
| `waterRipple` | `#b3f0ff` | Блики на воде |
| `beach` | `#cdb35b` | Пляж/песок |
| `riverbank` | `#905e24` | Берег реки (грязь/земля) |
| `grass` | `#80af49` | Трава (основная поверхность) |
| `underground` | `#1b0d03` | Подземелья |

## 3. Генерация террейна (пошагово)

### 3.1 Контур берега (Shore)

Берег — это неровный прямоугольник, отступающий от краёв карты.

**Алгоритм:**

1. Определить AABB берега:
   ```
   min = (shoreInset, shoreInset)                    // (48, 48)
   max = (width - shoreInset, height - shoreInset)   // (672, 672)
   ```

2. Разбить каждую сторону AABB на `N = 64` сегмента.

3. Для каждой точки деления добавить случайное смещение `[-shoreVariation, +shoreVariation]` (= 3 единицы) перпендикулярно стороне:
   - Нижняя и верхняя стороны: смещение по Y
   - Левая и правая стороны: смещение по X

4. Обход точек **против часовой стрелки**: нижняя сторона (лево->право) -> правая (низ->верх) -> верхняя (право->лево) -> левая (верх->низ).

5. Угловые точки НЕ смещаются (остаются на месте).

**Результат:** замкнутый полигон `shore[]` из ~260 точек.

### 3.2 Контур травы (Grass)

Трава — это береговой контур, сдвинутый к центру карты.

**Алгоритм:**

```
center = (width / 2, height / 2)

для каждой точки shore[i]:
    direction = normalize(center - shore[i])
    offset    = grassInset + random(-grassVariation, +grassVariation)   // 18 +/- 2
    grass[i]  = shore[i] + direction * offset
```

**Результат:** полигон `grass[]` повторяющий форму берега, но меньше. Пространство между `shore` и `grass` — это пляж.

### 3.3 Генерация рек

#### 3.3.1 Выбор конфигурации рек

Система использует взвешенный случайный выбор из пресетов:

| Вес | Конфигурация |
|-----|-------------|
| 0.10 | 1 река шириной 4 |
| 0.25 | 2 реки: ширина 8 и 4 |
| 0.20 | 3 реки: ширина 16, 8, 4 |
| 0.15 | 3 реки: ширина 8, 6, 4 |
| ... | и т.д. |

#### 3.3.2 Генерация точек реки

1. Выбрать случайную начальную и конечную точки на противоположных краях карты.
2. Рекурсивное подразделение (5-6 проходов):
   - Взять середину между двумя соседними точками.
   - Сместить перпендикулярно на `random * (расстояние_между_точками / 7)`.
   - Направление смещения случайно (влево/вправо).
3. Результат сглаживается в catmull-rom сплайн с параметром `smoothness = 0.45`.
4. Итого ~33 точки на реку.

#### 3.3.3 Генерация полигонов реки

Для каждой реки из сплайна создаются два полигона:

```
shoreWidth = clamp(riverWidth * 0.75, 4.0, 8.0)
```

Для каждой точки сплайна:
- Вычислить нормаль (перпендикуляр к касательной)
- `waterPoly`:  точка +/- normal * waterWidth
- `shorePoly`:  точка +/- normal * (waterWidth + shoreWidth)

**Расширение на концах** (чтобы река красиво впадала в океан):
```
end = max(1 - i/len, i/len) - 0.5      // 0 в центре, 0.5 на краях
waterWidth_local = (1.0 + end^3 * 1.5) * baseWidth
```

#### 3.3.4 Озёра (looped rivers)

Озеро — это замкнутая река. Точки расположены по кругу:
- `innerRad = 30`, `outerRad = 50`
- Смещение идёт радиально от центра озера, а не по нормали сплайна
- Вероятность появления: `odds = 0.5`

### 3.4 Ground Patches (цветные патчи на земле)

Патчи — это цветные прямоугольники с рваными краями, привязанные к зданиям.

**Параметры патча:**

| Поле | Тип | Описание |
|------|-----|----------|
| `min`, `max` | Vec2 | AABB патча (мировые координаты) |
| `color` | number | Цвет заливки |
| `roughness` | float | Плотность неровностей (0.0 - 1.0+) |
| `offsetDist` | float | Амплитуда неровностей |
| `order` | 0 или 1 | Порядок отрисовки (0 = под сеткой, 1 = над сеткой) |
| `useAsMapShape` | bool | Показывать ли на миникарте |

**Формула деления для неровных краёв:**
```
divisionsX = round((width  * roughness) / offsetDist)
divisionsY = round((height * roughness) / offsetDist)
```

Затем к каждой промежуточной точке на стороне добавляется смещение `[-offsetDist, +offsetDist]`.

## 4. Порядок отрисовки (снизу вверх)

Это критически важный порядок. Каждый следующий слой рисуется поверх предыдущего.

```
Слой 1:  CLEAR COLOR = grass (зелёный) — фон всего экрана

Слой 2:  BACKGROUND — четыре полосы по 120 ед. за пределами карты
          цвет: background (тёмно-синий)

Слой 3:  BEACH — полигон shore[] с "дыркой" grass[] внутри
          цвет: beach (песочный)
          Технически: заливаем shore, вырезаем hole = grass

Слой 4:  GRASS (только в canvas-mode) — полигон grass[]
          В WebGL-mode трава = clear color, виден через "дырку"

Слой 5:  GROUND PATCHES (order = 0)
          Цветные патчи с рваными краями (под сеткой)

Слой 6:  RIVER SHORES — полигоны river.shorePoly для каждой реки
          цвет: riverbank (коричневый)

Слой 7:  RIVER WATER — полигоны river.waterPoly для каждой реки
          цвет: water (синий)

Слой 8:  OCEAN — прямоугольник всей карты с "дыркой" shore[]
          цвет: water
          Это заливает всё вокруг острова водой

Слой 9:  GRID — линии через каждые gridSize (16) единиц
          цвет: чёрный, alpha = 0.15
          толщина: 2 / pixels_per_unit (масштабируется с камерой)

Слой 10: GROUND PATCHES (order = 1)
          Патчи поверх сетки

Слой 11: GAME OBJECTS — деревья, камни, ящики, здания
          Каждый объект со своим z-index
```

## 5. Размещение объектов на карте

### 5.1 Порядок спавна

Объекты размещаются в два этапа:

**Этап 1 (важные объекты):**
1. `locationSpawns` — объекты с фиксированными позициями (нормализованные 0-1, умноженные на размер карты) и радиусом допуска
2. `importantSpawns` — ключевые здания (больше попыток размещения)
3. Мосты через реки (размер моста зависит от ширины реки)

**Этап 2 (массовый спавн):**
1. `fixedSpawns` — фиксированное количество: `{ warehouse_01: 2, hut_01: 3 }`
2. `randomSpawns` — выбрать N из M: `{ spawns: [A, B, C], choose: 2 }`
3. `densitySpawns` — количество пропорционально площади карты:
   ```
   count = densityValue * (mapArea / 250000)
   // Пример: stone_01: 350 при карте 720x720 = 350 * (518400/250000) ~ 726 камней
   ```

### 5.2 Правила размещения

При размещении каждого объекта проверяется:
- Не пересекается с уже размещёнными объектами (коллизия)
- Не попадает в воду (реку/океан) — если объект не водный
- Не попадает в запретные зоны (маски рек)
- Объект имеет ориентацию `ori` (0-3 = 0/90/180/270 градусов) и масштаб

### 5.3 Мосты

Мосты размещаются на пересечении рек с условными линиями:

| Ширина реки | Тип моста | Макс. количество |
|-------------|-----------|-----------------|
| 5-8 | medium | 3 |
| 9-19 | large | 2 |
| 20+ | xlarge | 0 (не ставятся) |

## 6. Миникарта

Миникарта рендерится в отдельную текстуру (`RenderTexture`) размером `screenHeight x screenHeight`.

### 6.1 Отрисовка миникарты

1. Залить фон цветом `grass`
2. Отрисовать террейн (тот же `renderTerrain`, но с флагом `mapRender = true`)
3. Отрисовать границу карты (чёрная линия)
4. Отрисовать объекты как упрощённые фигуры:
   - Здания: цветные прямоугольники/круги по их коллайдерам
   - Препятствия: цветные круги/AABB
   - Z-сортировка по `zIdx`
5. Отрисовать названия мест (`PIXI.Text`, белый, bold Arial, alpha=0.75)

### 6.2 Упрощённые формы объектов

Для каждого объекта на миникарте:
```
shape = {
    collider: Circle или AABB,
    color: hex цвет (из определения объекта),
    scale: множитель размера (обычно 1.0)
}
```

Здания могут определять кастомные `map.shapes[]`, иначе используется их `ceiling.zoomRegions[0]` или bounding collider.

## 7. Камера и трансформация

Карта рисуется в мировых координатах. Для отображения на экране:

```
screenPoint = camera.pointToScreen(worldPoint)

// Упрощённо:
screenX = (worldX - camera.posX) * pixelsPerUnit + screenWidth / 2
screenY = -(worldY - camera.posY) * pixelsPerUnit + screenHeight / 2
```

Ось Y инвертирована (в мире Y вверх, на экране Y вниз).

Контейнер `ground` просто двигается и масштабируется каждый кадр:
```
p0 = camera.pointToScreen(0, 0)
p1 = camera.pointToScreen(1, 1)
scale = p1 - p0

ground.position = p0
ground.scale = scale
```

## 8. Алгоритм "рваных краёв" (Jagged AABB)

Ключевой визуальный элемент — все природные формы имеют неровные края.

**Вход:** AABB (min, max), divisionsX, divisionsY, variation, seededRandom

**Алгоритм:**
```
points = []

// Нижняя сторона (лево -> право)
points.push(ll)  // угол без смещения
for i = 1 to divisionsX:
    x = ll.x + spanX * i
    y = ll.y + random(-variation, +variation)
    points.push(x, y)

// Правая сторона (низ -> верх)
points.push(lr)
for i = 1 to divisionsY:
    x = lr.x + random(-variation, +variation)
    y = lr.y + spanY * i
    points.push(x, y)

// Верхняя сторона (право -> лево)
points.push(ur)
for i = 1 to divisionsX:
    x = ur.x - spanX * i
    y = ur.y + random(-variation, +variation)
    points.push(x, y)

// Левая сторона (верх -> низ)
points.push(ul)
for i = 1 to divisionsY:
    x = ul.x + random(-variation, +variation)
    y = ul.y - spanY * i
    points.push(x, y)
```

Где:
```
spanX = (max.x - min.x) / (divisionsX + 1)
spanY = (max.y - min.y) / (divisionsY + 1)
```

## 9. Seeded Random

Вся генерация использует детерминированный PRNG, инициализированный seed'ом карты. Это гарантирует что:
- Все клиенты видят одинаковую карту
- Рваные края одинаковы у всех игроков
- Можно воспроизвести карту по seed'у
